# SystemSculpt 5.3.1

## What's New

SystemSculpt 5.3.1 is a reliability release for the Pi-backed chat experience and the mobile release lane around it. The main work here is tightening the places where the plugin could look healthy in development but fail in the real runtime: blank hosted continuations in chat, Android sync accidentally pushing a dev bundle, and release verification missing the exact lane users depend on.

## Highlights

- Hosted chat turns now recover more cleanly when the upstream model finishes with hidden reasoning but no visible assistant output, instead of silently leaving behind a blank turn.
- Empty hosted continuations after tool execution now retry a few times and then fail loudly with a real error path, which is safer than pretending the turn succeeded.
- Local Pi execution imports are now loaded lazily, reducing startup-time coupling and making the service layer safer to import in non-runtime contexts.
- Android plugin sync now builds and verifies a production artifact set by default, and it rejects `main.js` bundles that still contain inline dev sourcemaps.
- The Android debug/open flow now supports a no-focus headless path, so real-device validation no longer has to steal the host machine’s window focus.
- Native runtime smoke is stricter and more realistic on mobile, including fresh chat leaves per case, retry handling for transient upstream rate limits, and stronger embeddings verification against freshly seeded fixtures.
- The local release script is more resilient when `GITHUB_TOKEN` or `GH_TOKEN` is weaker than the stored `gh` login; push and draft-release steps now fall back automatically when the stored auth has the right scope.

## Important Notes

- This is a patch release. The main user-facing changes are chat reliability and Android/mobile release safety, not a new product surface.
- The Android load failure investigated in this release pass came from syncing an oversized dev watcher bundle into the mobile vault. The release/test tooling now blocks that path by default.
- The current tree is fully validated locally, but there are still no committed changes after tag `5.3.0`, so an actual release still requires one clean release commit before the standard tag-and-draft workflow can run.
- No new API migration is required for users in this release candidate.

## Pi, Chatview, and Mobile Release Work Included

- chatview hosted-tool continuation recovery for reasoning-only empty responses
- explicit failure handling for unrecoverable empty hosted turns
- lazy-loading of local Pi execution modules for safer service imports
- import-safety test coverage for the Pi execution service boundary
- production-artifact enforcement for Android sync and debug-open flows
- headless Android launch support for no-focus verification
- stronger mobile smoke coverage for fresh chat leaves, embeddings processing, hosted auth bootstrap, and transient rate-limit retries
- release-script auth fallback when env-provided GitHub credentials are weaker than stored `gh` auth
- internal video-studio compatibility shims to stay aligned with the plugin surface used for release/demo assets
