# SystemSculpt 6.2.0

SystemSculpt 6.2.0 brings a new Audio Processor to your vault. Choose existing audio or paste a YouTube link, then let SystemSculpt create a structured audio note and a linked, timestamped transcript without asking you to configure prompts or summary settings.

- Audio Processor accepts vault and device audio as well as public YouTube videos. Uploads continue on the SystemSculpt service, so long recordings do not need to keep Obsidian busy.
- Completed notes include source attribution, a concise summary, key points, decisions, action items, discussion topics, risks, blockers, and open questions when the source supports them. Empty sections are left out instead of filled with guesses.
- Multilingual transcription and speaker-aware audio processing are handled automatically. YouTube citations link back to the matching timestamp.
- Interrupted work can be resumed safely, and the transcript remains recoverable even if a later summary step cannot finish.
- Chat and Studio now use the current SystemSculpt AI model configuration while preserving the existing OpenRouter and vault-tool architecture.
- The unused legacy Workflow Automations settings, commands, and backlog screens are gone. Inbox audio transcription and Studio workflows remain available.
- Audio selection, recording recovery, local note delivery, and mobile behavior have additional safeguards for stale files, interrupted sessions, and duplicate completion attempts.
