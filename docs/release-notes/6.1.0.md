# SystemSculpt 6.1.0

SystemSculpt 6.1.0 makes the plugin materially safer on mobile and turns Studio projects into stable, agent-editable vault files instead of fragile private state.

- Settings, Chat, Similar Notes, Studio, modals, and fixed surfaces now share one adaptive mobile contract with safer host-capability checks, owned mobile chrome handling, better spacing, and portable fallbacks instead of desktop-only dead ends.
- `Copy Current File Path` now ships with `Mod+Shift+C`, prefers the actually focused vault-backed surface, and falls back to a vault-relative path on mobile hosts where absolute filesystem paths do not exist.
- `.systemsculpt` files are now the single editable Studio project source of truth. Generic vault file tools can edit them directly, while private Studio persistence paths remain guarded.
- Studio now validates agent-written project documents before write, keeps stable generated fields protected, and rejects malformed or unsafe project mutations before they hit disk.
- Project sessions are more resilient under autosave, reload, rename, and external-edit races. When a file edit beats an in-memory canvas save, Studio preserves the losing canvas as recovery/Undo instead of silently dropping work or overwriting the file.
- Studio image and media editing is less brittle: the inline image toolbar lives with the image surface, crop removal/deselection is explicit, and history/file-edit synchronization is better covered by tests.
