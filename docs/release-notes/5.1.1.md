# SystemSculpt 5.1.1

## What's New

SystemSculpt 5.1.1 focuses on the reliability issues users feel first: mobile transcription parity, clearer large-upload failures, and a release path grounded in the native Obsidian runtime instead of older drift-prone test infrastructure.

This release also finishes the move to the shared native smoke harness as the canonical integration path for shipping the plugin, so the lane we maintain is the same lane we trust for release verification.

## Highlights

- SystemSculpt transcription now uses the hosted jobs pipeline on mobile as well as desktop, bringing the two environments into closer parity.
- Audio uploads now make chunking decisions based on estimated multipart request size rather than raw blob size alone, which better protects near-limit recordings from provider-side size failures.
- Plain-text `413` upload errors now surface directly instead of being wrapped in a confusing JSON parse failure.
- The shared `testing/native/` runtime smoke harness is now the canonical release-validation path, replacing the older WDIO-era testing lane and retiring stale release plumbing that no longer matched the real plugin workflow.

## Important Notes

- `5.1.0` remains an unpublished historical draft state. `5.1.1` is the public release built from the current verified plugin tree.
- Desktop release validation passed through the extended native runtime smoke lane, including chat, file read/write, embeddings, transcription, record-and-transcribe, web fetch, and YouTube transcript coverage.
- Android native runtime smoke passed the early extended checks used in this release pass, but the Android embeddings validation lane still needs follow-up and remains a known gap rather than a claimed green lane.
- Windows support remains documented, but Windows has not yet received a fresh live host validation pass for this release.

## Verification

The `5.1.1` release candidate passed the following release gates on the committed tree:

- `git diff --check`
- `npm run check:plugin:fast`
- `npm test`
- focused transcription test coverage for hosted jobs parity, multipart-aware chunking, and plain-text `413` handling
- desktop extended native runtime smoke
- `node scripts/release-plugin.mjs --dry-run --version 5.1.1`

Desktop extended native runtime smoke covered:

- exact-answer chat
- file read
- file write
- embeddings
- transcription
- record and transcribe
- live web fetch
- YouTube transcript retrieval

## API and Platform Work Included

- Hosted SystemSculpt transcription jobs now apply consistently on mobile and desktop.
- Upload handling now supports desktop file handles when available and vault-backed bytes when running on mobile.
- Custom-provider chunking now uses estimated multipart request size rather than only the raw blob size.
- Provider upload failures now preserve clearer plain-text error messages when that is what the upstream service returns.
- Native runtime smoke under `testing/native/` is now the maintained release contract for integration verification.
