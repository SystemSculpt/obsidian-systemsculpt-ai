# SystemSculpt 5.8.1

## What's New

SystemSculpt 5.8.1 is an OpenAI provider setup patch. Adding an OpenAI API key now makes GPT-5.4 Mini available in the Chat model picker, and chat requests for it route to OpenAI using your key.

This continues the provider setup work from 5.7.1, which fixed the same kind of "provider connected but its model is missing from Chat" gap for xAI.

## Highlights

- A connected OpenAI provider now adds GPT-5.4 Mini to the Chat model picker.
- Chat requests for that model route to OpenAI's API with your own key.
- Added a guard so a configured provider's model cannot silently disappear from the picker again.

## Upgrade Notes

- No manual action is needed.
- If you already added an OpenAI key but its model did not appear, update the plugin, then refresh provider status or reconnect OpenAI.
- Minimum supported Obsidian version remains `1.4.0`.

## Fixes and Reliability

- Fixed OpenAI bring-your-own-key models not appearing in the Chat model dropdown after the provider was configured.
- Added a regression guard that checks every model offered in the picker has a working execution endpoint, so this class of dropdown gap is caught before release.

## Verification

Plugin checks, unit tests, and the built-bundle checks passed locally. The changes in this release also passed the macOS desktop baselines and the Windows clean-install and desktop-baselines end-to-end checks in continuous integration. Android and iOS device validation were not run for this patch.
