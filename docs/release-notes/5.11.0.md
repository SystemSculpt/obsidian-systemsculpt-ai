# SystemSculpt 5.11.0

## What's New

SystemSculpt 5.11.0 makes Studio text nodes feel native to Obsidian and restores
Chat Resend for desktop users. Text nodes now use Obsidian's Markdown editor,
including live-preview formatting and task controls, while preserving the caret
and edit history through Studio graph updates.

## Highlights

### Studio text editing

- **Native Markdown editing.** Studio text nodes now use Obsidian's embedded
  Markdown surface instead of a plain textarea, so headings, emphasis, code,
  links, lists, and task checkboxes behave like they do in a note.
- **Stable focus through graph updates.** Pointer placement, selection, scroll
  position, and focus survive the rerenders that happen while working in a
  Studio project.
- **Reliable save and history behavior.** Leaving a text node flushes the latest
  edit, a continuous editing session is grouped into one undo step, and deleting
  an empty node remains undoable and redoable.
- **Safer editor teardown.** Studio releases Obsidian's editor, focus, and keymap
  ownership even when a final commit or snapshot fails.

### Chat

- **Resend works again on desktop.** Resending from an earlier message no longer
  fails with `Failed to fetch dynamically imported module: node:fs`. Desktop
  filesystem access now stays behind the plugin's desktop-only boundary, with a
  built-bundle regression check to prevent the failure from returning.

## Upgrade Notes

- No manual action is required.
- Minimum supported Obsidian version remains `1.4.0`.

## Verification

The release passed the plugin checks, full unit suite, built-bundle integration
suite, and the Linux, macOS, and Windows GitHub end-to-end lanes. Native Studio
editing and Chat Resend were also verified in the packaged plugin inside
Obsidian.
