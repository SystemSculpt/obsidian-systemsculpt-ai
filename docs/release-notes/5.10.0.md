# SystemSculpt 5.10.0

## What's New

SystemSculpt 5.10.0 is a focused security, privacy, and reliability release. It
came out of a deep audit of the plugin and closes a cluster of correctness gaps:
a secret no longer reaches a debug log, a syncable policy can no longer smuggle a
blanket command-approval wildcard, rate limits now degrade gracefully, semantic
search can no longer silently misalign, and settings failures are no longer
swallowed. Every change shipped failing-test-first with a permanent CI guard, so
the behaviors stay fixed.

## Highlights

### Security & privacy

- **License key kept out of logs.** The chat debug request log previously
  captured the full request headers, including your SystemSculpt license key.
  The debug payload is now redacted; the real request is unchanged.
- **No blanket command approval via sync.** A Studio agent policy can no longer
  carry a bare `*` wildcard that would auto-approve every CLI command. The
  wildcard is stripped when policies are parsed, ignored by the approval gate,
  and scrubbed from any policy already on disk.
- **License validity downgrades only on an authoritative rejection.** A real
  `401`/`403` from the license service now correctly marks the license invalid,
  while transient/network errors preserve your last-known-good state instead of
  locking you out on a blip.

### Reliability

- **Rate limits degrade gracefully.** The client now honors the server's
  `Retry-After` on a managed `429`, classifies bring-your-own-key `429`s as
  transient, and performs a bounded, abort-aware retry — with a guarantee that
  already-streamed content can never be replayed. This is the client complement
  to the server-side OpenRouter hardening.
- **Streams stop when a chat view closes.** Closing a chat now aborts its
  in-flight request instead of leaving the network stream, token spend, and
  render loop running in the background.
- **Honest edit-tool results.** The filesystem agent's edit tool now reports
  zero or partial edits accurately instead of claiming success when nothing (or
  only some) of the requested changes applied, and it skips no-op writes.
- **Settings failures are surfaced.** If saving (or backing up) settings fails —
  disk full, permissions, a sync lock — the error is now logged and shown once
  via a de-duplicated notice, instead of being silently swallowed and lost on
  reload.

### Semantic search

- **Embedding vectors stay aligned with their chunks.** Whitespace-only or empty
  inputs no longer shift the returned vectors out of alignment with their source
  chunks, which could silently corrupt semantic-search results. The provider now
  returns one slot per input and asserts the counts match.

### Developer experience

- **The CSS linter is now actually enforced.** The Obsidian-override hygiene
  linter that ships with the repo is wired into the check gate (locally and in
  CI), with a guard test proving it fails on an unscoped selector — so the rule
  it was written to catch is finally guaranteed.
- Removed a dead token-counter code path (and its orphaned support files) that
  referenced a dependency that was never installed.

## Upgrade Notes

- No manual action is required; settings migrate automatically on update.
- Minimum supported Obsidian version remains `1.4.0`.

## Verification

Every change in this release landed through its own pull request with the full
CI matrix green — the unit suite and built-bundle integration suite, plus the
macOS and Windows end-to-end lanes — and an automated review pass. Each fix
added or tightened a permanent test guard at the cheapest layer that catches the
regression. The release additionally ran the native release matrix: macOS
desktop baselines, the Windows clean-install and desktop-baselines GitHub
end-to-end check for the released commit, and the Android sync and runtime smoke
lanes.
