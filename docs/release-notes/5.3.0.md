# SystemSculpt 5.3.0

## What's New

SystemSculpt 5.3.0 focuses on the Pi-powered chat experience and the native desktop automation lane we use to prove it. The main theme is reliability through simplification: cleaner chatview model switching, better transcript continuity when moving between SystemSculpt and local Pi, and a no-focus Obsidian automation path that is sturdier under reload churn.

## Highlights

- Chatview model switching is cleaner and easier to maintain, with the picker flow split into clearer ownership boundaries instead of more logic living inline in the main view.
- Transcript sync now preserves the non-Pi portion of a conversation when switching between SystemSculpt and local Pi, instead of collapsing the visible chat down to only the newest Pi turn.
- The desktop automation bridge now waits for a stable bridge generation, recovers more cleanly from bridge rollovers, and has dedicated chatview stress coverage for repeated model-switch and reload churn.
- Duplicate embeddings status bars are now cleaned up during reloads so repeated development or automation cycles do not leave the footer stacked with stale entries.
- External `data.json` sync now has a polling fallback in addition to `fs.watch`, so the no-focus bridge recovery path is less dependent on one watcher mechanism behaving perfectly on the live desktop host.

## Important Notes

- This release is intended to ship using the native no-focus desktop automation path under `testing/native/`, not the older renderer-driving desktop approach.
- The new external-settings polling fallback requires one manual plugin reload in an already-open vault the first time this build is introduced. After that, the steady-state goal is fully no-focus recovery and verification.
- Local automated verification is already green on the current tree, and the live attach-only desktop revalidation is now green on the already-open target vault after a clean quit/reopen.
