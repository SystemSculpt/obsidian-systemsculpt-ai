# SystemSculpt 5.3.3

## What's New

SystemSculpt 5.3.3 is a chat reliability patch focused on two failure shapes in the assistant timeline. The visible fix is that reasoning, tool calls, and content now stay in true emitted order while a response is streaming and after a saved chat is reopened. The deeper fix is that managed SystemSculpt turns no longer lose the final answer when hosted tool-call ids repeat across continuation rounds.

## Highlights

- ChatView now preserves chronological assistant output instead of collapsing reasoning blocks together or bunching tool calls away from the content they belong to.
- Persisted chats now reload in the same order they were emitted, so a saved conversation keeps its reasoning, tool-use, and response fragments aligned.
- Managed SystemSculpt continuations now remap repeated hosted tool-call ids per assistant occurrence, which prevents later tool results from attaching to the wrong round and restores final answer delivery for long tool-driven turns.
- The regression surface is now locked down with targeted coverage for renderer order, reasoning layout, markdown round-trips, hosted continuation remapping, and desktop automation reopen flows for saved chat history.
- Release preflight is also cleaner in this patch line: the Windows runtime version fixture no longer contains a tracked machine-specific path that would block a local release.

## Important Notes

- This is a patch release focused on correctness and continuity in ChatView. It does not introduce a new top-level product surface.
- No manual migration is expected. Existing chats and saved history should simply render and continue more reliably after updating.
- The missing-final-answer bug was isolated to the managed SystemSculpt hosted transcript path. A provider-backed parity rerun on `local-pi-openrouter@@openai/gpt-5.4-mini` still completed successfully, which is why the fix is scoped to hosted continuation remapping instead of changing the Pi/provider execution path.
- The native release matrix passed on macOS, Windows, and Android during this release pass. iOS runtime smoke remains optional and unavailable in this environment because no physical iOS device was attached.

## API and Platform Work Included

- Assistant timeline rendering now keeps reasoning, tool calls, and content in the same chronological order across live streaming and saved-chat reloads.
- Chat markdown persistence now round-trips interleaved assistant parts without flattening them into the wrong sequence on reopen.
- Managed hosted continuation requests now allocate unique API tool-call ids for repeated upstream tool ids and bind tool messages back to the correct assistant occurrence.
- Desktop automation helpers can now reopen saved chat history directly, which gives this failure mode a durable live repro and regression-proof path.
- Release safety preflight stays green against tracked machine-specific Windows test fixtures, keeping the local `gh` release lane shippable.
