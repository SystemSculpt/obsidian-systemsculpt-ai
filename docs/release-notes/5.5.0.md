# SystemSculpt 5.5.0

## What's New

SystemSculpt 5.5.0 is a Studio, Chat, and Pi-provider reliability release. The headline change is the new Studio canvas foundation: large graph workspaces now have explicit viewport math, expanded bounds, redesigned node-card rendering, and sturdier connection geometry.

This release also tightens the provider setup path and chat failure experience. Local Pi providers are easier to understand from Settings, OAuth and API-key state is surfaced more clearly, provider disconnects are more reliable, and streaming failures now produce actionable notices instead of leaving the conversation in an unclear state.

## Highlights

- Studio now has the foundation for near-infinite canvas work, with larger navigable bounds, dedicated viewport math, cleaner graph-node rendering, and connection handling built for bigger layouts.
- Chat now explains streaming failures more clearly and preserves a better recovery path when provider errors or disconnect states interrupt a response.
- The Providers tab now distinguishes subscription login, saved API keys, and local Pi `models.json` setup more clearly, including local Ollama guidance.
- Pi auth inventory and storage paths are more resilient when bundled auth storage is unavailable, when local provider credentials live in plugin settings, or when desktop reload timing makes direct storage access unreliable.
- The local development loop now has a tracked `run.sh` entrypoint for build watching, plugin sync, and hot reload into configured vault targets.
- Claude PR review automation was hardened with clearer review instructions, larger review budget, JSON validation for findings, and safer workflow behavior around tokens and changed-file analysis.

## Important Notes

- This is a minor release because the Studio canvas foundation and visible provider/chat UX changes are more than a patch-level maintenance update.
- No manual vault migration is expected. Existing chats, provider settings, and Studio data should continue to load normally.
- Local provider setup still depends on the Pi `models.json` configuration path. The Providers tab now explains that path more directly instead of treating local providers like hosted subscription accounts.
- The Claude PR review workflow changes are release-engineering improvements. They do not change plugin behavior inside Obsidian, but they improve review quality for future work.

## API and Platform Work Included

- Studio graph viewport math now lives in a dedicated module and has focused tests around canvas bounds.
- Studio graph node-card rendering is split into a renderer module, giving the visual redesign a clearer ownership boundary.
- Pi auth inventory now falls back cleanly when the bundled auth storage cannot initialize.
- Provider setup UI now handles local provider configuration separately from OAuth and saved API-key flows.
- Chat streaming errors are normalized through a dedicated utility and surfaced through a modal flow.
- Direct remote provider chat requests now share the Chat Completions message normalizer used by managed requests, including safe tool-call id remapping for follow-up turns.
- The release script now supports a pre-bumped release candidate, so version metadata can be reviewed before tagging without forcing the publish step to bump again.
