# SystemSculpt 5.5.0

## What's New

SystemSculpt 5.5.0 improves Studio, chat reliability, and provider setup. The Studio canvas now has a stronger foundation for larger graph workspaces, with expanded navigation bounds, steadier viewport behavior, cleaner node-card rendering, and connection handling built for bigger layouts.

Provider setup is clearer too. The Providers tab now does a better job separating subscription login, saved API keys, and local Pi provider setup, including local Ollama guidance. Chat also recovers more clearly when streaming or provider errors interrupt a response.

## Highlights

- Studio now supports much larger graph workspaces with improved canvas bounds, viewport behavior, node-card rendering, and connection handling.
- Chat now gives clearer feedback when streaming fails or a provider connection needs attention.
- Provider setup now explains local Pi configuration, saved API-key state, and subscription login state more directly.
- Pi provider authentication is more resilient when local auth storage is temporarily unavailable.
- Direct provider agent/tool turns are more reliable after tool calls, especially for Chat Completions-compatible providers.
- Release and review automation received behind-the-scenes hardening for safer future updates.

## Upgrade Notes

- No manual vault migration is expected.
- Existing chats, provider settings, and Studio data should continue to load normally.
- Minimum supported Obsidian version remains `1.4.0`.
- Local Pi provider setup still depends on the configured Pi `models.json` path.

## Fixes and Reliability

- Improved chat recovery after streaming, quota, auth, and provider disconnect errors.
- Improved provider disconnect persistence and cleanup behavior.
- Improved local provider setup tests and fallback behavior for Pi auth inventory.
- Improved release tooling so reviewed version metadata can be prepared before tagging.
- Improved automated review validation used during development.
