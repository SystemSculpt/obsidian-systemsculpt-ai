# SystemSculpt 5.8.0

## What's New

SystemSculpt 5.8.0 adds image generation to the desktop Studio. A Studio image node turns a text prompt into images through SystemSculpt's hosted image service, with direct control over the model, aspect ratio, image size, seed, and how many images to generate per run.

This release also removes the legacy CanvasFlow feature, which decorated Obsidian's native canvas and was superseded by Studio.

## Highlights

- Generate images from a Studio image node: choose the model, aspect ratio, image size, seed, and batch count (up to 4).
- Leave the seed blank for random results, or set a whole number for reproducible output.
- The model picker only lists models that can actually generate images, so you never select one the service rejects.

## Upgrade Notes

- No manual action is needed.
- CanvasFlow has been removed; the desktop Studio view replaces it. Any saved `canvasFlowEnabled` setting is cleaned up automatically on upgrade.
- Image generation is a desktop-only Studio feature.
- Minimum supported Obsidian version remains `1.4.0`.

## Fixes and Reliability

- Image generation requests normalize cleanly: camelCase and snake_case option names are both accepted, and an image size the chosen model does not support is dropped instead of failing the request.
- Idempotent retries of an image job stay stable across the image-size change, so a repeated request replays the original job instead of returning a conflict.
- Selecting an unsupported image model now returns a clear invalid-model response.

## Verification

Plugin checks, unit tests, release surface checks, the macOS desktop baselines, the Windows clean-install and desktop-baselines GitHub check, and the Android sync + runtime smoke passed before release. iOS device validation was not run for this release.

## API and Platform Work Included

- Backend image-generation request schema hardening (camelCase option aliases, image-size-aware idempotency hashing, explicit invalid-model errors), deployed to api.systemsculpt.com ahead of this release.
- Removed the legacy CanvasFlow modules and relocated the shared image-generation modules to `src/services/images/`.
