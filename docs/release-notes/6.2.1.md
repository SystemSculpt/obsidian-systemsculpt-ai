# SystemSculpt 6.2.1

SystemSculpt 6.2.1 fixes Audio Processor uploads on Obsidian desktop. The plugin now lets the host set the binary request length, avoiding a restricted-header failure that could stop vault and device audio before it reached the SystemSculpt service.

- Vault and device audio can upload through Audio Processor on desktop without the `net::ERR_INVALID_ARGUMENT` failure.
- Signed binary uploads validate their expected byte length before sending while leaving the host-owned request header to Obsidian.
- The same transport safeguard covers Audio Processor, audio transcription, document processing, and image input uploads.
- YouTube processing, multilingual transcription, speaker-aware summaries, durable recovery, and credit settlement continue to use the server-owned 6.2 pipeline.
