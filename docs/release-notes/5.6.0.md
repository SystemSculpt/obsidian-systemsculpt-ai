# SystemSculpt 5.6.0

## What's New

SystemSculpt 5.6.0 is a focused Search release. The in-vault Search modal now opens faster, keeps results steady while the content index catches up, and handles punctuated, multilingual, and symbol-only queries that used to drop out of results. Recent notes show up immediately for empty queries, and their snippets stay fresh after you edit a file.

If you use Obsidian's "Excluded files" filters, or you have embeddings enabled, this release also fixes several cases where Search used to disagree with your current settings until you reloaded.

## Highlights

- Search now opens with an instant recents list and starts returning metadata matches before the full content index finishes building, so the first results land almost immediately.
- Search now stays stable while the content index refreshes, so notes you are reading stop popping in and out of results between keystrokes.
- Search now finds punctuated queries like `orange-juice`, `don't`, and `co-op` against notes that contain the exact punctuated phrase, instead of silently dropping those hits after the index is ready.
- Search now handles multilingual and symbol-only queries more reliably, including CJK (e.g. `東京`), Cyrillic, and emoji-only terms.
- Recent notes now pick up fresh previews after an edit instead of showing a stale snippet until the next full search.
- Search now respects Obsidian's "Excluded files" filter the moment you change it, both for recents and for live search results.
- Smart mode now lazily starts the embeddings manager when embeddings are enabled, so users with auto-processing off still get semantic hits without a manual reload.

## Upgrade Notes

- No manual action is needed.
- Existing chats, settings, and search behavior should continue to work as before.
- Minimum supported Obsidian version remains `1.4.0`.

## Fixes and Reliability

- Fixed a case where changing Obsidian's "Excluded files" filter only took effect for the first change, leaving later edits ignored until an unrelated vault event.
- Fixed a case where starting a smart search during a background index build could leave excluded files searchable even after the filter had changed.
- Fixed short prefix queries (e.g. typing `fr` on the way to `fresh`) so they keep returning matches after the content index is ready.
- Fixed the empty-query recents list so it drops newly excluded notes and shows updated previews for recently modified files.
- Fixed a case where smart mode silently fell back to lexical-only when embeddings were enabled but the embeddings manager had not been initialized yet.
