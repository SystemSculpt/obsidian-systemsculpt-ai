# SystemSculpt 6.0.0

SystemSculpt 6 replaces the old client-owned provider stack with one managed, cross-device architecture.

- Chat, transcription, embeddings, and Studio use SystemSculpt services with no provider-key setup.
- Chat is rebuilt from the ground up as a native agent workspace with streaming tool progress, inline approvals, durable artifacts, retries, cancellation, and queued follow-ups.
- Provider catalogs, model selection, local Pi execution, custom endpoints, and third-party tool-server configuration were removed.
- Existing managed chats continue normally. Chats tied to retired Pi or external-provider metadata open read-only.
- Settings migration removes retired credentials while preserving the SystemSculpt license, vault folders, Chat favorites, Studio projects, recorder preferences, embeddings, and workflows.
- The plugin supports desktop and mobile; local CLI, FFmpeg, and external-file Studio operations remain desktop-only.
