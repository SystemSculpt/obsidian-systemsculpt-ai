# SystemSculpt 5.7.0

## What's New

SystemSculpt 5.7.0 is a Studio and chat reliability release. You can now paste or drag media files straight into Studio, so screenshots and images land on the canvas without a detour through your vault. Long, tool-heavy chats are also better at recovering their final answer when a turn gets interrupted mid-stream.

Managed models now pick up their names, limits, and capabilities from the SystemSculpt server, so the model list stays current without waiting for a plugin update.

## Highlights

- Studio now accepts pasted and dropped media files directly onto the canvas, including images from your clipboard.
- Chat now recovers interrupted turns more reliably, so long tool-driven replies are less likely to lose the final answer after a stream hiccup.
- Chat keeps its streaming indicator consistent through repeated tool calls in a single turn, so the view reflects what the model is actually doing.
- Managed models now sync their metadata from the server, so new models and updated limits appear without a plugin update.
- Support links now point to the SystemSculpt support page.

## Upgrade Notes

- No manual action is needed.
- Existing chats, settings, and Studio boards continue to work as before.
- Minimum supported Obsidian version remains `1.4.0`.

## Fixes and Reliability

- Fixed cases where an interrupted chat turn could leave the conversation stuck instead of resuming or surfacing the partial reply.
- Hardened tool-loop handling so repeated tool calls in one turn keep the streaming indicator and final message consistent.
- Model metadata now falls back to the bundled catalog when the server is unreachable, so offline use keeps working.

## Verification

Plugin checks, the unit test suite, and the release surface gate passed before tagging.
